# 人物画像分析 Skill

用于生成并维护 `resources/<talker>/profile.json` 中的人物画像。

## 输入要求

- 必须基于 WeFlow 读取到的真实聊天记录。
- 推荐读取方式：`node .opencode/skills/weflow/weflow.mjs messages --talker <会话ID> --limit 10000`
- 分析对象默认是对方，不是“我”。需要区分：
  - `isSend = 1` 或 `direction = me`：我方发言
  - `isSend = 0` 或 `direction = other`：对方发言
- 可以补充双方关系画像，但不能把我方发言误判成对方人格。

## 分析原则

1. 只基于真实消息做判断；不能凭空推断职业、关系、性格或经历。
2. 不把一次性玩笑、表情包、情绪化短句或孤立事件过度解释成稳定特征。
3. 每个重要判断都要给出证据来源：可以是简短摘录，也可以是聊天内容概括。
4. 明确区分强信号、弱信号和不确定项。
5. 避免诊断式标签，避免把对方固定成单一性格。
6. 画像要服务于后续聊天理解：保留可帮助理解语境、关系、称呼、梗、边界和偏好的信息。

## 输出格式

必须输出一个合法 JSON 对象，不要输出 JSON 之外的解释文字。

```json
{
  "schemaVersion": 1,
  "displayName": "<显示名>",
  "talker": "<会话ID>",
  "generatedAt": "<ISO时间>",
  "summary": "<一句话概览>",
  "basicInfo": [
    {
      "key": "age",
      "label": "年龄",
      "value": "未知 / 约20岁 / 明确年龄",
      "confidence": "unknown",
      "evidence": "为什么这样判断；没有证据就写未知"
    }
  ],
  "analysisMarkdown": "## 性格与沟通分析\n\n..."
}
```

### basicInfo 字段要求

`basicInfo` 是前端清晰展示的基本角色信息区。必须覆盖以下维度，未知也要输出，`value` 写“未知”，`confidence` 写 `unknown`。前端会隐藏 `unknown` 字段，所以不要为了凑字段编造信息：

- `age` / 年龄
- `occupation` / 职业/学习状态
- `education` / 学校/专业/学习线索
- `location` / 常见地点/活动范围
- `importantPeople` / 经常出现的重要人物
- `interests` / 兴趣偏好

每个字段都要包含：

- `key`：英文稳定键名
- `label`：中文标签
- `value`：短文本，适合前端卡片展示
- `confidence`：只能是 `high`、`medium`、`low`、`unknown`
- `evidence`：简短证据；没有证据就说明“聊天记录中无明确证据”

### analysisMarkdown 字段要求

`analysisMarkdown` 是长文本分析区，使用 Markdown。这里可以写得详细，重点承载性格、关系、行为模式等复杂分析。每个主要部分建议至少 2-4 段，并尽量包含“观察 → 证据 → 解释 → 边界/不确定性”的推理链。不要只写短 bullet。

建议包含以下部分：

```md
## 1. 性格与沟通风格

- 表达习惯
- 回复节奏
- 玩笑/调侃方式
- 冲突、回避或缓和方式
- 证据
- 对稳定模式和偶发情境分别说明

## 2. 兴趣与关注点

- 高频主题
- 主动提起的话题
- 长期重复出现的偏好
- 证据
- 哪些偏好只是当前阶段的弱信号

## 3. 情绪与压力模式

- 常见情绪
- 压力来源
- 求助、安慰、吐槽方式
- 证据
- 情绪波动中的触发因素和缓和方式

## 4. 行动与决策模式

- 计划方式
- 执行倾向
- 约定落实情况
- 证据
- 对时间、金钱、社交约定、风险的处理方式

## 5. 双方关系画像

- 亲疏程度
- 信任程度
- 调侃边界
- 常见互动剧本
- 关系中的敏感点
- 证据
- 双方在互动中的角色分工和默契

## 6. 重要记忆点

- 可帮助后续对话理解的事实、称呼、梗、偏好、禁区

## 7. 不确定项

- 哪些判断只是弱信号
- 还需要哪些聊天记录验证

## 8. 后续相处建议

- 具体、克制、可执行的沟通建议
```

长度要求：`analysisMarkdown` 应该是详尽分析，不少于 1200 个中文字符；如果聊天记录很少，也要明确说明材料不足，并把不确定项写清楚。

## 质量标准

- 详尽但克制。
- 不要写成泛泛的星座式性格报告。
- 不要只给结论，要给证据。
- 不要只总结事件，要抽象出长期模式。
- 对敏感内容保持中性描述，不做道德审判。
