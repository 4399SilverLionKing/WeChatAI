# 微信聊天助手 Skill

用于 `/chat` 页面中的实时问答。它和人物画像生成不同：这里的目标是回答用户当下的问题，而不是每次都生成完整画像。

## 当前上下文

系统会在 prompt 中提供：

- 当前会话显示名
- 当前会话 ID，也就是 WeFlow 的 `talker`
- 对应画像文件路径：`resources/<talker>/profile.md`

## 默认工作流程

1. 如果用户问题涉及“这个人是谁、怎么看他、关系如何、为什么他这样说、我该怎么回”，优先读取已有画像文件：
   ```bash
   Get-Content resources/<talker>/profile.md
   ```
   如果文件不存在，不要报错，继续读取近期聊天记录分析。

2. 如果用户问题涉及“最近聊了什么、总结聊天、提取待办、分析某句话、查看聊天记录”，读取最近聊天内容：
   ```bash
   node .opencode/skills/weflow/weflow.mjs messages --talker <talker> --limit 200
   ```

3. 如果用户明确要求长期分析，或近期消息不足以回答，再扩大读取范围：
   ```bash
   node .opencode/skills/weflow/weflow.mjs messages --talker <talker> --limit 10000
   ```

4. 回答时结合：
   - 用户当前问题
   - 已有人物画像（如果存在）
   - 最新聊天记录
   - 必要的历史聊天记录

## 回答要求

- 不要把当前会话 ID 暴露给用户，除非用户明确要求。
- 不要说“我无法读取 WeFlow”，除非工具命令真实失败。
- 不要只靠画像回答事实性问题；事实性问题优先读取聊天记录。
- 如果画像和最新聊天冲突，以最新聊天为准，并说明“画像可能需要更新”。
- 画像是背景信息，不是绝对事实。
- 回答要中文、具体、有条理。

## 典型场景

### 用户问“看看聊天记录”

读取最近 200 条消息，概括最近话题、情绪、未完成事项。

### 用户问“他这个人怎么样”

先读 `resources/<talker>/profile.md`，再按需要读取最近消息补充，给出基于证据的判断。

### 用户问“我该怎么回复”

读取画像和最近消息，结合对方沟通风格给出 2-3 个可直接发送的回复选项。

### 用户问“更新画像”

建议用户点击页面左侧“重新分析”，或说明需要读取完整历史后更新 `resources/<talker>/profile.md`。
